## Feedback Synthesis Standards

**Feedback Collection Sources:**
- App Store and Play Store reviews
- Customer support tickets and conversations
- In-app feedback and surveys
- Social media mentions and comments
- User interviews and research sessions
- Community forums and discussion boards
- Analytics behavior data
- NPS and CSAT surveys

**Pattern Identification:**
- Cluster similar feedback themes
- Identify frequency of issues
- Assess severity and user impact
- Track sentiment (positive, neutral, negative)
- Spot emerging trends early
- Connect feedback to user segments

**Analysis Framework:**
1. **Collect:** Aggregate from all sources
2. **Categorize:** Group by theme/feature
3. **Quantify:** Count occurrences and impact
4. **Contextualize:** Add user segment and behavior data
5. **Prioritize:** Rank by impact √ó frequency
6. **Recommend:** Actionable product changes

**Feedback Categories:**
- **Bugs/Issues:** Things that are broken
- **Feature Requests:** Things users want
- **UX Complaints:** Confusing or frustrating experiences
- **Performance:** Speed and reliability concerns
- **Pricing:** Cost-related feedback
- **Competitive:** Mentions of competitors
- **Praise:** What's working well

**Sentiment Analysis:**
- Track overall sentiment trends
- Identify sentiment by feature
- Monitor sentiment changes over time
- Correlate sentiment with events (releases, etc.)
- Segment sentiment by user type
- Flag sudden sentiment drops

**Prioritization Matrix:**
```
High Impact, High Frequency: Fix immediately (P0)
High Impact, Low Frequency: Address soon (P1)
Low Impact, High Frequency: Batch fix or improve (P2)
Low Impact, Low Frequency: Backlog or defer (P3)
```

**User Quote Mining:**
- Extract verbatim quotes for context
- Highlight especially articulate feedback
- Use quotes in internal documentation
- Share quotes with team for empathy
- Include in product specs
- Feature in marketing (with permission)

**Competitive Intelligence:**
- Track mentions of competitor features
- Identify competitive advantages/weaknesses
- Monitor switching reasons
- Analyze competitor review themes
- Identify market gaps

**Behavioral Correlation:**
- Link feedback to user behavior data
- Identify patterns in complaint origins
- Understand feedback in context of usage
- Segment feedback by user cohorts
- Correlate feedback with retention

**Feedback Response Strategy:**
- Acknowledge all feedback
- Respond to critical issues publicly
- Show users their feedback matters
- Communicate product changes
- Close the feedback loop
- Turn critics into advocates

**Metrics to Track:**
- Feedback volume by source
- Sentiment scores over time
- Most mentioned features/issues
- Feature request frequency
- Resolution rate for reported issues
- Time to acknowledge/resolve
- NPS and CSAT trends

**Tools and Systems:**
- Feedback aggregation platforms
- Sentiment analysis tools
- Review monitoring services
- Support ticket analytics
- Survey platforms
- Social listening tools
- Text analysis and tagging systems

**Synthesis Cadence:**
- **Daily:** Critical issues and urgent patterns
- **Weekly:** Theme analysis and trend spotting
- **Monthly:** Comprehensive report and roadmap input
- **Quarterly:** Major trends and strategic recommendations

**Translating Feedback to Features:**
- Identify underlying needs, not just stated wants
- Look for patterns across multiple users
- Consider technical feasibility
- Validate with usage data
- Prototype and test solutions
- Measure impact after implementation
