## UX Research Standards

**Rapid Research Methodologies:**
- Design guerrilla research for quick insights
- Create micro-surveys (< 5 questions)
- Conduct remote usability tests efficiently
- Use analytics data to inform qualitative research
- Develop research plans that fit development cycles
- Extract actionable insights within days

**User Journey Mapping:**
- Create detailed journey maps with emotional touchpoints
- Identify critical pain points and moments of delight
- Map cross-platform user flows
- Highlight drop-off points with data
- Design intervention strategies
- Prioritize improvements by impact

**Behavioral Analysis:**
- Track user interactions and patterns
- Identify where users struggle
- Understand mental models
- Analyze task completion rates
- Study decision-making patterns
- Discover unmet needs

**Persona Development:**
- Create data-driven user personas
- Include goals, motivations, frustrations
- Add behavioral patterns and preferences
- Update personas with new insights
- Use personas to guide decisions
- Make personas actionable, not decorative

**Usability Testing:**
- Design efficient test protocols (30-45 min sessions)
- Recruit representative users quickly
- Conduct moderated or unmoderated tests
- Focus on critical user flows
- Identify usability issues systematically
- Provide actionable recommendations

**Research Methods:**

**Qualitative:**
- User interviews (1-on-1)
- Focus groups
- Usability testing
- Diary studies
- Contextual inquiry
- Think-aloud protocols

**Quantitative:**
- Surveys and questionnaires
- A/B testing analysis
- Analytics review
- Heat maps and session recordings
- Card sorting
- Tree testing

**User Interview Protocol (45 min):**
1. **Introduction (5 min):** Build rapport, explain purpose
2. **Context Questions (10 min):** Background and current behavior
3. **Tasks (15 min):** Observe actual usage, note pain points
4. **Reflection (5 min):** Gather feelings, uncover desires
5. **Wrap-up (3 min):** Final thoughts, next steps

**Research Sprint Timeline (1 week):**
- **Day 1-2:** Plan research, recruit participants
- **Day 3-4:** Conduct research sessions
- **Day 5-6:** Analyze findings, identify patterns
- **Day 7:** Present insights and recommendations

**Data Analysis:**
- Look for patterns across participants
- Identify common pain points
- Prioritize by frequency and severity
- Create actionable recommendations
- Link findings to business metrics
- Visualize insights for stakeholders

**Research Insights Documentation:**
- Executive summary (key findings)
- Detailed findings with quotes/evidence
- User journey maps
- Pain point prioritization
- Recommendations with rationale
- Next steps and success metrics
