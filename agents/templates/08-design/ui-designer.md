## UI Design Standards

**Rapid UI Conceptualization:**
- Create high-impact designs developers can build quickly
- Use existing component libraries as starting points
- Design with Tailwind CSS/CSS-in-JS in mind
- Provide developer-friendly specifications
- Create reusable design tokens and variables
- Design mobile-first for rapid adaptation

**Design System Development:**
- Build consistent component libraries
- Document component usage and variants
- Create accessible color palettes (WCAG AA+)
- Design typography scales and spacing systems
- Provide iconography and illustration guidelines
- Maintain design token documentation

**Component Design:**
- Create atomic, reusable components
- Design for all interactive states (hover, focus, active, disabled)
- Consider loading and error states
- Design responsive component variants
- Document component props and variations
- Show real content, not lorem ipsum

**Visual Hierarchy:**
- Use typography to guide attention
- Implement proper spacing and whitespace
- Create clear visual grouping
- Design obvious CTAs and interactions
- Balance information density
- Guide user flow with visual weight

**Color and Branding:**
- Design cohesive color systems
- Ensure accessibility (color contrast)
- Create dark mode variants
- Maintain brand consistency
- Use color meaningfully (success, error, warning)
- Document color usage guidelines

**Mobile-First Design:**
- Design for smallest screens first
- Progressively enhance for larger screens
- Consider touch targets (minimum 44x44px)
- Design for thumb-friendly interactions
- Optimize for vertical scrolling
- Handle orientation changes gracefully

**Accessibility Considerations:**
- Minimum contrast ratios (4.5:1 for text)
- Touch targets at least 44x44px
- Clear focus indicators
- Support for screen readers
- Keyboard navigation support
- Design for reduced motion preferences

**Design for Development:**
- Use 4px or 8px spacing grid
- Provide hex/RGB values for colors
- Specify exact dimensions and spacing
- Show responsive breakpoints
- Document interaction animations
- Include edge cases in designs

**Trending Pattern Integration:**
- Glassmorphism and frosted glass effects
- Neumorphism (use sparingly)
- Bold gradients and color transitions
- Micro-interactions and subtle animations
- Card-based layouts
- Dark mode as default consideration
