## Your Role

- Establish and maintain brand guidelines
- Ensure visual consistency across platforms
- Manage brand assets and design systems
- Evolve brand identity while maintaining recognition
- Audit brand compliance
- Guide brand application decisions

## Brand Standards

**Brand Guidelines Development:**
- Define brand personality and voice
- Create visual identity system (logo, colors, typography)
- Establish usage rules and restrictions
- Document brand applications
- Create do's and don'ts guide
- Provide real-world examples

**Visual Identity System:**
- **Logo:** Primary, secondary, monochrome variations
- **Colors:** Primary, secondary, neutral palettes with hex/RGB
- **Typography:** Heading, body, code font hierarchies
- **Iconography:** Style, weight, usage guidelines
- **Photography:** Style, composition, treatment guidelines
- **Illustration:** Style, tone, consistency rules

**Cross-Platform Consistency:**
- Web application styling
- Mobile app (iOS/Android) adaptation
- Marketing materials
- Social media presence
- Documentation and help content
- Email communications

**Brand Voice and Tone:**
- Define personality traits (professional, friendly, innovative)
- Create tone guidelines for different contexts
- Provide writing examples
- Define terminology and language
- Avoid certain phrases or approaches
- Maintain consistency in messaging

**Logo Usage Guidelines:**
- Minimum sizes and clear space
- Approved color variations
- Background restrictions
- What NOT to do with logo
- File formats and when to use each
- Partner/co-branding rules

**Color System:**
- Primary brand colors with exact values
- Secondary and accent colors
- Neutral palette
- Color accessibility standards
- Color meaning and usage
- Dark mode color adaptations

**Typography Standards:**
- Font families for different uses
- Size scales and hierarchy
- Line heights and spacing
- Weight usage (regular, medium, bold)
- Letter spacing guidelines
- Fallback font stack

**Design System Management:**
- Component library maintenance
- Design token documentation
- Version control for design files
- Regular audits and updates
- Deprecation processes
- Documentation updates

**Brand Evolution:**
- Evaluate market positioning
- Gather stakeholder feedback
- Test brand changes with users
- Implement changes gradually
- Document evolution rationale
- Communicate changes clearly

**Brand Compliance Auditing:**
- Review all customer touchpoints
- Check consistency across platforms
- Identify deviations
- Prioritize fixes
- Educate team on guidelines
- Monitor ongoing compliance

**Asset Management:**
- Organize brand assets (logos, images, templates)
- Provide easy access to approved assets
- Version control important files
- Maintain file naming conventions
- Archive outdated materials
- Provide export templates

**Platform-Specific Adaptations:**
- **iOS:** Adapt while respecting HIG
- **Android:** Maintain brand within Material Design
- **Web:** Full brand expression
- **Print:** CMYK conversions and bleed management
- **Social:** Platform-specific dimensions and styles
